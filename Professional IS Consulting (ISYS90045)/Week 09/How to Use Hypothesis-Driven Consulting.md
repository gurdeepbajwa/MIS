# How to Use Hypothesis-Driven Consulting

Course: [[Professional IS Consulting (ISYS90045)/Course Overview|Professional IS Consulting (ISYS90045)]]
Week: 09
Source: [[Professional IS Consulting (ISYS90045)/Week 09/Lecture|Week 09 Lecture]]
Diataxis type: How-to guide

## Purpose

Use hypothesis-driven consulting to focus analysis in ambiguous client problems while staying open to evidence that changes the recommendation.

## When to Use It

Use it when:

- the client problem is broad or unclear;
- time and budget do not allow every issue to be analysed equally;
- the consultant needs to prioritise evidence collection;
- the answer may change as facts emerge.

Do not use it as a way to defend a preferred answer. A hypothesis is a testable starting point, not a conclusion.

## Steps

1. Clarify the decision the client needs to make.
2. Define the problem tightly enough to guide analysis.
3. Separate symptoms from possible causes.
4. Develop one or more provisional hypotheses.
5. Break each hypothesis into issue-tree questions or evidence needs.
6. Prioritise hypotheses by potential impact and uncertainty.
7. Gather evidence through data, interviews, observation, documents, workshops, or analysis.
8. Test whether the evidence supports, weakens, or changes the hypothesis.
9. Revise the issue tree and hypothesis when evidence changes.
10. Communicate the current answer first, then the reasons, evidence, assumptions, and limits.

## Hypothesis Pattern

```markdown
We believe that [proposed cause or action] for [stakeholder or business area] will [expected outcome].
We will validate this when [evidence or result] shows [business impact].
```

## Common Mistakes

| Mistake | Better Practice |
| --- | --- |
| Treating the first hypothesis as the answer | Treat it as a learning tool. |
| Collecting evidence only to confirm the preferred view | Actively look for disconfirming evidence. |
| Analysing everything equally | Prioritise high-impact and high-uncertainty areas. |
| Hiding hypothesis changes from the client | Explain how evidence changed the view. |
| Sounding overconfident too early | State assumptions, confidence, and evidence gaps. |

## Exam Use

In an exam answer, hypothesis-driven consulting is useful for showing structured judgement. A strong answer should mention:

- decision clarity;
- problem framing;
- issue trees or smaller questions;
- evidence testing;
- revision;
- client communication.

## Retrieval Prompts

1. Why is hypothesis-driven consulting not the same as assuming the answer?
2. What makes a hypothesis testable?
3. How does hypothesis-driven work reduce wasted analysis?
4. How should a consultant explain a changed hypothesis to the client?
