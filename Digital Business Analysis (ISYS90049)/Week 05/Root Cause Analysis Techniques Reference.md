## Purpose

This reference compares the Week 05 problem-analysis techniques so they can be selected deliberately rather than used interchangeably.

## Core Distinctions

| Technique | Best For | Output | Main Risk |
| --- | --- | --- | --- |
| Five-Why method | Tracing a narrow causal chain from symptom to deeper cause. | A possible root-cause chain. | Oversimplifying complex or multi-cause problems. |
| Fishbone diagram | Organising many possible causes across categories. | A cause-and-effect map. | Producing a large list without evidence or prioritisation. |
| Pareto analysis | Prioritising causes by frequency, impact, or contribution. | A ranked view of the vital few causes. | Treating frequency as importance without checking business impact. |

## Common Sequence

1. Define the problem.
2. Use elicitation, documents, observation, or data to identify possible causes.
3. Use a fishbone diagram if the causes span multiple categories.
4. Use Five Whys to probe important causal paths.
5. Use Pareto analysis where cause frequency or impact data is available.
6. Translate the stronger causes into requirements, solution options, corrective actions, or further investigation.

## Cause Categories

Fishbone diagrams may use the 6M categories when appropriate:

| Category | Typical DBA Meaning |
| --- | --- |
| Machine | Systems, integration, interfaces, infrastructure, automation. |
| Method | Workflow, procedures, policies, handoffs, communication. |
| Material | Data, documents, inputs, physical materials, source quality. |
| Man | Skills, workload, training, motivation, role clarity. |
| Measurement | Reporting, calculation, estimation, KPI design, data interpretation. |
| Milieu | Environment, context, dependencies, culture, constraints. |

## Exam Distinctions

- Five Whys finds a possible causal chain.
- Fishbone organises possible cause categories.
- Pareto ranks causes once there is evidence about frequency or impact.
- None of the techniques replaces stakeholder engagement or evidence validation.

## Retrieval Prompts

1. When would Pareto analysis be inappropriate?
2. Why might a fishbone diagram be better than Five Whys for a cross-functional issue?
3. What evidence would strengthen a root-cause claim?
4. How do root-cause findings affect later requirements?
